Bridge Jira & Git
from your terminal.

DevTrace eliminates context-switching by auto-formatting commits, auto-posting updates to Jira, and providing a unified interface to view and manage tickets — all without leaving your terminal.

bash — devtrace
$ devtrace start DT-21
→ Active ticket set to DT-21

$ git commit -m "implement api"
→ DT-21 | FEAT : implement api
→ Posted to Jira: Automated DevTrace Update

$ devtrace tkt DT-21
→ Fetching ticket details…

Configuration

All credentials and state are stored in a local TOML file. Never transmitted to DevTrace servers.

Config location

~/.devtrace/configs/local/local_config.toml

Config structure

[jira]
host = "https://your-org.atlassian.net"
email = "your@example.com"
api_token = "ATATT3xFfGF0_..."

[active]
ticket_id = "DT-21"
branch = "main"
started_at = "2026-02-21T18:44:05+05:30"
Getting a Jira API Token: Visit id.atlassian.com → Security → API tokens, click Create API token, and paste it during devtrace init jira.

CLI Commands

Click any command to expand its full reference documentation.

devtrace init
Initialize DevTrace in a repository
▾

Creates the .devtrace/ folder and activates Git hooks in the current repository.

devtrace init

What it does:

  • Creates .devtrace/ directory in the project root
  • Installs Git hooks: prepare-commit-msg, commit-msg, post-commit
  • Sets core.hooksPath to .devtrace/hooks
devtrace init jira
Configure Jira credentials interactively
▾

Prompts for your Jira host URL, email, and API token, then validates the connection.

devtrace init jira
PromptExample
Jira Hosthttps://your-org.atlassian.net
Emailyour@example.com
API TokenATATT3xFfGF0_…
devtrace start <ticket-id>
Set the active Jira ticket for the session
▾

All subsequent commits and Jira comments will be linked to this ticket automatically.

devtrace start DT-21
devtrace start PROJ-456
ArgumentTypeRequiredDescription
ticket-idstringYesThe Jira ticket ID (e.g. DT-21)
Only one ticket can be active at a time. Use devtrace start <new-id> to switch mid-session.
devtrace tickets
List assigned Jira tickets in a rich table
▾

Supports filtering by status and limiting results. Output includes ticket ID, title, status, dates, and direct Jira links.

# All open tickets
devtrace tickets

# Filter by status
devtrace tickets list --status "In Progress"

# Limit + filter
devtrace tickets list --status "To Do" --limit 10
FlagTypeDefaultDescription
--statusstringAllFilter by status (e.g. "In Progress")
--limitintegerAllLimit the number of returned tickets
devtrace tkt <ticket-id>
View full details for a specific ticket
▾

Displays a rich terminal panel with description, assignee, status, and recent comments.

devtrace tkt DT-21
devtrace tkt DT-21 --no-comments
FlagTypeDefaultDescription
--no-commentsflagFalseHide the comments section from output
devtrace comment "<message>"
Post a comment to a Jira ticket
▾

Defaults to the active ticket. Use --ticket to target a specific one.

devtrace comment "Working on this now"
devtrace comment "Looks good!" --ticket DT-25
Argument / FlagRequiredDescription
messageYesComment text (wrap in quotes)
--ticketNoTarget a specific ticket ID
If no active ticket and no --ticket is provided, the command will error and prompt you to run devtrace start <ticket-id>.

Git Hooks

Three hooks are installed automatically by devtrace init. They live in .devtrace/hooks/ and run during the commit lifecycle.

1
prepare-commit-msg
Trigger: before the commit message editor opens
Reads the active ticket ID, detects commit type from keywords, and injects the formatted prefix automatically.
# Input
implement api endpoint

# Output
DT-21 | FEAT : implement api endpoint
2
commit-msg
Trigger: after the commit message is written
Validates the commit message against the required format. Rejects any commits that don't match the pattern.
# Valid format
[TICKET-ID] | [TYPE] : [Description]

✓ DT-21 | FEAT : implement Jira integration
✗ feat: implement api (missing ticket ID)
Bypass with: git commit --no-verify -m "WIP: quick save" — use sparingly.
3
post-commit
Trigger: immediately after a successful commit
Auto-posts a structured comment to the active Jira ticket with the commit hash and files changed. Silently skips on WIP commits or if Jira is unreachable. Never blocks the commit.
# Auto-posted Jira comment
🤖 Automated DevTrace Update: Code committed
Commit Hash: a3f8b2c
Files Changed: +45, -3
# Disable for one commit
git commit -m "[WIP] DT-21 | CHORE : quick save"

Commit Message Format

All commits in a DevTrace-enabled repository must follow this structure exactly.

[TICKET-ID] | [TYPE] : [Description]
PartDescriptionExample
TICKET-IDYour Jira ticket identifierDT-21, PROJ-456
TYPEOne of the allowed commit typesFEAT, FIX
DescriptionShort description of the changeimplement login flow

Allowed Commit Types

FEATNew feature or functionality
FIXBug fix
INITInitial setup or scaffolding
DOCSDocumentation changes only
REFACTORRestructuring without behavior change
TESTAdding or updating tests
CHOREBuild, CI/CD, dependency updates

Troubleshooting

Jira credentials not found
▾
devtrace init jira
Git hooks not running
▾
# Verify hooks path
git config --get core.hooksPath
# Expected: .devtrace/hooks

# Re-activate if missing
git config core.hooksPath .devtrace/hooks
Auto-comments not posting to Jira
▾

Check these in order:

  1. No active ticket — Run devtrace start DT-21
  2. Bad credentials — Run devtrace init jira again
  3. WIP commit — Hooks skip [WIP] commits by design
  4. Network/Jira down — Check your Jira host URL is reachable

Workflow Examples

Standard patterns for using DevTrace in daily development.

Standard Daily Workflow

# 1. View all open tickets
devtrace tickets

# 2. Pick and activate a ticket
devtrace start DT-21

# 3. Stage and commit (hooks auto-format + auto-post)
git add src/
git commit -m "implement feature"
# → Transformed to: DT-21 | FEAT : implement feature
# → Auto-posted to Jira: Automated DevTrace Update

# 4. Check progress + post manual update
devtrace tkt DT-21
devtrace comment "Ready for code review"

WIP Save (Skip Jira Posting)

git commit -m "[WIP] DT-21 | CHORE : checkpoint save"
# → Committed locally, NOT posted to Jira

Emergency Commit (Skip All Hooks)

git commit --no-verify -m "hotfix: critical production issue"
# → Skips formatting, validation, and Jira posting
